# План: как начать зарабатывать на ML одному

> [!warning] О данных
> План опирается на локальные исследования [[ml-market-research-2026]], [[niche-1-llm-fintech-legal]], [[niche-2-cv-industrial-medical]] и [[niche-3-asr-tts-voice]], но адаптирован под режим **один человек без команды**.
> Рыночные цифры в исходных исследованиях — оценки порядка величины. Перед конкретным коммерческим предложением перепроверь цены API, GPU и локальные ставки на дату сделки.

## Главный принцип для соло

Один человек не может конкурировать с командами в «больших» проектах (обучение фундаментальных моделей, MLOps-платформы, длинные enterprise-внедрения). Соло выигрывает там, где:

1. **Короткий цикл «задача → результат → деньги»** (дни–недели, не месяцы).
2. **Узкая повторяемая задача**, которую можно превратить в шаблон/продукт.
3. **Готовые модели + дообучение/интеграция**, а не обучение с нуля.
4. **Чёткий ROI для клиента** — он видит, на чём экономит/зарабатывает.

> [!tip] Стратегия «лестница»
> Услуги (фриланс, быстрые деньги) → Продуктизация (повторяемый пакет) → Микро-SaaS/инфопродукт (пассивный доход).
> Не пытайся прыгнуть сразу на последнюю ступень — начни зарабатывать услугами, на них же изучи реальные боли клиентов.

## Сравнение трёх ниш для соло

| Критерий | LLM финтех/легал | CV пром/мед | ASR/TTS голос |
|---|---|---|---|
| Порог входа (навыки) | Средний | Высокий | Средний |
| Нужны свои данные | Часто (клиентские) | Да, много | Иногда (open-source хватает) |
| Нужно дорогое железо (GPU) | Низко (API + RAG) | Высоко (обучение CV) | Средне (дообучение/инференс) |
| Размер чека | Высокий | Высокий | Средний |
| Скорость до первых денег | Быстро | Медленно | Средне |
| Конкуренция | Высокая | Средняя | Средняя |
| Регуляторика/риски | Высокие (compliance, мед/фин) | Высокие (мед — сертификация) | Низкие–средние |
| Подходит соло на старте | **Да** | Частично | **Да** |

### Ниша 1 — LLM для финтеха/легала
**Что продавать соло:** RAG-системы по внутренним документам (договоры, регламенты, базы знаний), ассистенты для поддержки, извлечение данных из документов, дообучение под терминологию домена, LLM-агенты для рутины.

- **Плюсы:** низкий порог по железу (часто хватает API + векторная БД), быстрый MVP, высокий чек, понятный ROI («экономим N часов юриста»).
- **Минусы:** высокая конкуренция, требования к точности и compliance (галлюцинации в легале/фине — недопустимы), нужно доверие клиента к данным.
- **Соло-вердикт:** **лучший старт по соотношению скорость/чек.** Минимум железа, максимум готовых инструментов.

### Ниша 2 — Computer Vision для промышленности/медицины
**Что продавать соло:** детекция дефектов на производстве, подсчёт/инвентаризация по фото, контроль качества, анализ снимков (с осторожностью — мед требует сертификации).

- **Плюсы:** меньше прямой конкуренции от LLM-вендоров, очень ощутимый ROI на производстве, чек высокий.
- **Минусы:** нужны данные (сбор/разметка), GPU для обучения, длинный цикл продажи (пилот на заводе = месяцы), мед — регуляторный ад для одиночки.
- **Соло-вердикт:** **сильная вторая ступень**, но не первый заработок. Стартуй с промышленности (не мед), с готовых детекторов (YOLO) + дообучение под клиента.

### Ниша 3 — ASR/TTS / голос
**Что продавать соло:** транскрипция и саммари звонков/встреч, голосовые боты, озвучка контента, дообучение распознавания под акцент/домен/терминологию, диаризация.

- **Плюсы:** много сильных open-source моделей (Whisper и аналоги) — можно дообучать без обучения с нуля, рынок растёт, легко сделать демо.
- **Минусы:** часть рынка занята крупными API, чек обычно ниже, чем в легале/фине.
- **Соло-вердикт:** **хороший старт для продуктизации** — легко превратить в повторяемый сервис (микро-SaaS на транскрипции/саммари).

## Рекомендация

> [!success] Маршрут для соло
> **Старт:** Ниша 1 (LLM финтех/легал) — услугами. Самый быстрый путь к первым деньгам с минимальным железом.
> **Параллельно учишься продуктизировать:** Ниша 3 (ASR/TTS) — на ней проще всего сделать повторяемый микро-сервис.
> **Когда появятся деньги/связи:** Ниша 2 (CV пром) — высокий чек, но требует данных, GPU и терпения.

Почему так: у тебя «база есть» (sklearn, базовый PyTorch). LLM/RAG-направление почти не требует обучения нейросетей с нуля — это интеграция + дообучение + инженерия данных, что освоить с твоего уровня реально за недели, а не месяцы. CV требует самого глубокого ML-скилла и железа — оставляем на потом.

## Конкретный первый оффер

Не продавай «обучение ML-моделей». Это звучит дорого, рискованно и непонятно для клиента. Первый оффер должен быть проще:

> **RAG-ассистент по документам компании за 2 недели:** загрузка документов, поиск по ним, ответы с источниками, простой API/UI, eval-набор из 20–50 вопросов и отчёт по качеству.

Почему это подходит соло:

- можно собрать на готовых моделях и API без дорогого GPU;
- результат показывается демо-роликом за 2–3 минуты;
- клиент понимает ценность через экономию времени на поиске и разборе документов;
- после 2–3 проектов превращается в шаблон.

Рабочие файлы под оффер:

## Дорожная карта (90 дней)

### Фаза 0 — Подготовка (недели 1–4, параллельно с [[course-ml-training|курсом]])
- [ ] Пройти модули 1–4 курса (база → первая дообученная модель → RAG).
- [ ] Собрать 1 публичный кейс-проект: RAG-ассистент по набору документов (демо + репозиторий + короткое видео).
- [ ] Завести минимальную витрину: GitHub с README-кейсом, профиль на 1–2 фриланс-площадках, короткий лендинг (одна страница: «делаю AI-ассистентов по вашим документам»).

### Фаза 1 — Первые деньги через услуги (недели 5–8)
- [ ] Определить узкое предложение (одно!): например «RAG-ассистент по внутренней документации за фикс-цену за 2 недели».
- [ ] Сделать 1–2 проекта дёшево/в портфолио ради отзывов и реальных болей.
- [ ] Зафиксировать прайс: пакет «диагностика» (платная, недорогая) → «внедрение» (основной чек) → «поддержка» (подписка).
- [ ] Цель фазы: первый платящий клиент.

### Фаза 2 — Повторяемость (недели 9–12)
- [ ] Из 2–3 выполненных проектов вынести общий шаблон (бойлерплейт RAG: загрузка → чанкинг → эмбеддинги → векторная БД → ответ с источниками).
- [ ] Упаковать в фиксированный пакет с понятной ценой и сроком.
- [ ] Запустить вторую ногу: микро-сервис на ASR/TTS (например, транскрипция+саммари звонков) как продукт с подпиской.
- [ ] Цель фазы: 2+ источника дохода, один из которых частично пассивный.

### Фаза 3 — Масштабирование без найма (после 90 дней)
- [ ] Поднять чек за счёт кейсов и отзывов.
- [ ] Автоматизировать рутину (свои внутренние агенты/скрипты).
- [ ] Выбрать: углубляться в дорогие enterprise-услуги (Ниша 1/2) или растить продукт (Ниша 3).

## Форматы монетизации (от быстрого к пассивному)

1. **Фриланс-услуги под задачу** — быстрые деньги, учат рынку. (старт)
2. **Фиксированные пакеты** — «продукт-как-услуга», предсказуемый доход.
3. **Подписка/поддержка** — повторяющийся доход с существующих клиентов.
4. **Микро-SaaS** — один узкий инструмент (напр. транскрипция+саммари), self-serve.
5. **Инфопродукт** — когда наберёшь экспертизу: гайд/шаблоны/мини-курс по своей нише.

> [!note] Антипаттерны для соло
> - Браться за «обучим свою большую модель» — это команда и бюджет.
> - Мед-CV без партнёра с сертификацией/доступом к данным — юридически тупик.
> - Распыляться на все три ниши сразу — выбери одну для первых денег.
> - Бесконечно «доучиваться» без выхода на клиентов — учись на реальных задачах (см. [[course-ml-training]]).
