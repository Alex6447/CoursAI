# Курс: Обучение ML-моделей → до уровня требований проекта

> [!info] Для кого этот курс
> Стартовый уровень: **«база есть»** — знаешь Python, делал учебные проекты на sklearn, трогал базовый PyTorch/обучение.
> Цель: дойти до состояния, когда можешь брать реальные задачи из [[solo-monetization-plan|плана заработка]] — прежде всего **LLM/RAG для финтеха/легала** (Ниша 1) и **ASR/TTS** (Ниша 3).
> Формат: каждый модуль = теория + **задание** + **критерий зачёта**. Не переходи дальше, пока критерий не выполнен.

> [!tip] Как проходить
> - Учись на коде, а не на видео. На каждый модуль — рабочий репозиторий/ноутбук.
> - Веди заметки об ошибках и решениях прямо в Obsidian (это твоя будущая база для контента).
> - Капстоун (модуль 10) — это твой первый кейс для портфолио. Держи его в голове с модуля 1.

## Как пользоваться курсом в проекте

Курс не про «стать ML-исследователем вообще». Его задача — довести тебя до состояния, когда ты можешь один собрать, измерить и продать первый узкий ML/LLM-сервис из [[solo-monetization-plan|плана монетизации]].

Рабочий цикл на каждый модуль:

1. Прочитал только нужную теорию.
2. Сделал задание в коде.
3. Записал артефакт в [[ml-training-practice-log|трекер практики]]: ссылка на репозиторий/ноутбук, метрика, ошибка, вывод.
4. Перенёс полезный кусок в [[rag-mvp-spec|MVP RAG-ассистента]], если он относится к капстоуну.

> [!warning] Ограничение
> Не переходи к коммерческой фазе только по ощущению «понял теорию». Переход разрешён, когда закрыты критерии модуля 10 и чек-лист в [[rag-mvp-spec#Критерии приёмки MVP]].

## Официальные опорные материалы

Используй эти источники как базу, а не случайные пересказы:

- [scikit-learn model evaluation](https://scikit-learn.org/stable/modules/model_evaluation.html) — метрики и scoring, чтобы не выбирать accuracy по привычке.
- [Hugging Face Transformers fine-tuning](https://huggingface.co/docs/transformers/training) — стандартный путь дообучения transformer-моделей.
- [Hugging Face PEFT](https://huggingface.co/docs/peft/index) — LoRA/QLoRA и parameter-efficient fine-tuning.
- [Hugging Face TRL SFT Trainer](https://huggingface.co/docs/trl/sft_trainer) — supervised fine-tuning для LLM.
- [LlamaIndex Python framework](https://developers.llamaindex.ai/python/framework/) — практический RAG-пайплайн по документам.
- [MLflow model evaluation](https://mlflow.org/docs/latest/ml/evaluation/) — фиксация экспериментов и оценка качества.

## Модуль 1 — Фундамент обучения: цикл, данные, метрики
**Цель:** уверенно понимать, что происходит при обучении любой модели, и не путаться в метриках/переобучении.

**Что изучить (закрыть пробелы):**
- Разбиение данных: train / validation / test, утечки данных (data leakage).
- Цикл обучения: forward → loss → backward → шаг оптимизатора; эпохи, батчи, learning rate.
- Переобучение/недообучение, регуляризация, ранняя остановка.
- Метрики: классификация (accuracy, precision/recall, F1, ROC-AUC), регрессия (MAE/RMSE), почему accuracy врёт на несбалансированных данных.

**Задание:**
- Возьми табличный датасет (например, отток клиентов или классификацию). Сделай честный train/val/test split, обучи модель (можно sklearn), построй кривые обучения, посчитай precision/recall/F1.
- Намеренно вызови переобучение и покажи это на графике, затем убери его (регуляризация/ранняя остановка).

> [!success] Критерий зачёта
> Можешь объяснить своими словами, почему модель переобучилась, и доказать это графиком train vs val. Выбрал метрику, адекватную задаче (не accuracy по умолчанию), и обосновал выбор.

## Модуль 2 — PyTorch на рабочем уровне
**Цель:** писать обучение «руками», а не копировать целиком — это нужно для дообучения в следующих модулях.

**Что изучить:**
- `Dataset` и `DataLoader` (свой кастомный Dataset).
- Ручной training loop: `optimizer.zero_grad()` → loss → `backward()` → `step()`.
- Работа с устройством (CPU/GPU), перенос тензоров, `model.train()` / `model.eval()`, `torch.no_grad()`.
- Сохранение/загрузка чекпойнтов, фиксация seed для воспроизводимости.
- Логирование метрик (хотя бы print + сохранение в файл; по желанию — TensorBoard/W&B).

**Задание:**
- Реализуй с нуля кастомный `Dataset` + `DataLoader` и обучи простую нейросеть (например, классификация изображений типа FashionMNIST или текстов).
- Добавь сохранение лучшего чекпойнта по val-метрике и возобновление обучения из чекпойнта.

> [!success] Критерий зачёта
> Твой training loop написан вручную, обучение воспроизводимо (один и тот же seed → один результат), есть сохранение/загрузка модели и логи метрик по эпохам.

## Модуль 3 — Transfer learning и fine-tuning
**Цель:** перестать обучать «с нуля» и научиться дообучать готовые модели — основной режим работы соло.

**Что изучить:**
- Что такое предобученная модель, transfer learning, заморозка/разморозка слоёв.
- Разница: feature extraction (заморозили backbone) vs полный fine-tuning.
- Подбор learning rate для дообучения, расписания LR.
- Введение в PEFT (parameter-efficient fine-tuning) и зачем нужен LoRA (детали — в модуле 6).

**Задание:**
- Возьми предобученную модель (для изображений — например ResNet; для текста — небольшую модель из Hugging Face) и дообучи под свою задачу на маленьком датасете.
- Сравни три режима: обучение с нуля / feature extraction / полный fine-tuning. Сведи метрики и время в таблицу.

> [!success] Критерий зачёта
> Показал в таблице, что дообучение предобученной модели бьёт обучение с нуля на малом датасете, и объяснил почему. Понимаешь, когда замораживать слои, а когда нет.

## Модуль 4 — NLP/LLM: токены, эмбеддинги, transformers
**Цель:** понять, как устроены текстовые модели, чтобы строить RAG и дообучать LLM.

**Что изучить:**
- Токенизация (subword/BPE), что такое токен и почему важна длина контекста.
- Эмбеддинги: векторное представление текста, семантическая близость (cosine similarity).
- Базовая идея transformer/attention (на уровне интуиции, без вывода формул).
- Инференс LLM: prompt, температура, top-p, стоимость по токенам; локальные модели vs API.
- Экосистема Hugging Face: `transformers`, `tokenizers`, загрузка моделей.

**Задание:**
- Прогони текст через токенизатор, посчитай число токенов, поэкспериментируй с длиной.
- Построй эмбеддинги для набора предложений и найди ближайшие по смыслу через cosine similarity.
- Запусти инференс небольшой LLM (локально или через API) и поиграй параметрами генерации.

> [!success] Критерий зачёта
> Можешь объяснить, что такое токен, эмбеддинг и attention «на пальцах», и кодом находишь семантически близкие тексты через эмбеддинги.

## Модуль 5 — RAG: ядро Ниши 1 (LLM финтех/легал)
**Цель:** собрать RAG-систему «вопрос-ответ по документам с источниками» — это твой главный продаваемый навык.

**Что изучить:**
- Архитектура RAG: загрузка → чанкинг → эмбеддинги → векторная БД → retrieval → генерация ответа с цитатами.
- Стратегии чанкинга (размер, перекрытие), влияние на качество.
- Векторные БД (например FAISS локально, либо managed-решение), индексирование и поиск top-k.
- Сборка промпта с контекстом, ответ со ссылками на источник, борьба с галлюцинациями.
- Базовая оценка RAG: релевантность retrieval, фактологичность ответа.

**Задание:**
- Построй RAG по набору PDF/текстов (например, набор регламентов или договоров — можно публичные).
- Реализуй ответ с указанием, из какого документа/фрагмента он взят.
- Проверь на 10–15 вопросах, где система ошибается, и улучши чанкинг/retrieval.

> [!success] Критерий зачёта
> Рабочий RAG отвечает на вопросы по твоим документам, **всегда показывает источник**, и ты измерил качество хотя бы на небольшом наборе вопросов. Это уже MVP, который можно показывать клиенту.

## Модуль 6 — Дообучение LLM: LoRA/QLoRA
**Цель:** дообучать LLM под домен/стиль/терминологию клиента на доступном железе.

**Что изучить:**
- Instruction tuning: формат датасета (инструкция → ответ).
- LoRA и QLoRA: почему это влезает в скромный GPU, какие гиперпараметры важны (rank, alpha).
- Библиотеки: `peft`, `transformers`, `trl` (или аналоги), квантизация.
- Подготовка и чистка датасета — обычно решает больше, чем гиперпараметры.
- Оценка дообученной модели: до/после, на отложенной выборке.

**Задание:**
- Собери небольшой instruction-датасет под узкую задачу (например, ответы в стиле/терминологии конкретного домена).
- Дообучи LLM через LoRA/QLoRA, сравни ответы базовой и дообученной модели на тестовых примерах.

> [!success] Критерий зачёта
> Дообучил LLM на своих данных через LoRA/QLoRA на доступном железе и показал измеримое улучшение на отложенной выборке. Понимаешь, когда дообучение оправдано, а когда хватает RAG/промпта (важно: часто RAG дешевле и достаточно).

## Модуль 7 — ASR/TTS: дообучение и пайплайн (Ниша 3)
**Цель:** сделать повторяемый голосовой сервис — основа для микро-SaaS из плана.

**Что изучить:**
- ASR: модели распознавания речи (например Whisper-семейство), метрика WER (word error rate).
- Дообучение/адаптация под акцент, домен, терминологию; диаризация (кто говорит).
- TTS: синтез речи, выбор голосов, качество.
- Пайплайн: аудио → транскрипция → постобработка → саммари (связка с LLM из модулей 4–5).

**Задание:**
- Сделай пайплайн «аудио → транскрипт → краткое саммари» на готовой ASR-модели.
- Измерь WER на нескольких записях, попробуй улучшить (предобработка аудио/дообучение под домен).

> [!success] Критерий зачёта
> Есть рабочий пайплайн транскрипции+саммари, ты посчитал WER и понимаешь, как его снижать. Это заготовка продаваемого сервиса.

## Модуль 8 — CV: детекция и дообучение (Ниша 2, опционально на старте)
**Цель:** дообучать детекторы под задачу клиента (дефекты, подсчёт объектов). Можно отложить — CV это третья ступень плана.

**Что изучить:**
- Задачи CV: классификация / детекция / сегментация — что когда.
- Готовые детекторы (например YOLO-семейство), дообучение на своих данных.
- Разметка данных (форматы, инструменты), аугментации.
- Метрики детекции: IoU, mAP.

**Задание:**
- Размечай небольшой датасет (50–200 изображений) под детекцию одного класса объектов.
- Дообучи готовый детектор, посчитай mAP, визуализируй предсказания.

> [!success] Критерий зачёта
> Дообученный детектор находит твой объект на новых изображениях, ты посчитал mAP и понимаешь, сколько данных/железа нужно под реальный заказ.

## Модуль 9 — Оценка качества и продакшн
**Цель:** превратить модель в то, за что платят: измеримое качество + работающий сервис.

**Что изучить:**
- Построение honest-оценки под задачу (offline-метрики, тестовые наборы, для LLM — eval-наборы).
- Инференс-сервис: обернуть модель в API (например FastAPI), базовая обработка ошибок и лимитов.
- Стоимость инференса (токены/GPU-время) и как считать цену для клиента.
- Минимальный мониторинг: логирование запросов/ответов, отслеживание деградации.
- Приватность данных клиента (особенно важно для финтех/легал).

**Задание:**
- Оберни RAG из модуля 5 (или ASR-пайплайн из модуля 7) в HTTP-API с одним эндпойнтом.
- Сделай страницу/таблицу с метриками качества и прикинь себестоимость одного запроса.

> [!success] Критерий зачёта
> Твоя модель доступна по API, есть измеримые метрики качества и расчёт себестоимости запроса. Можешь назвать клиенту цену не «с потолка».

## Модуль 10 — Капстоун: кейс под Нишу 1 (портфолио)
**Цель:** собрать законченный кейс, который станет твоим первым коммерческим предложением.

**Что сделать (объединяет модули 4, 5, 9):**
- Выбери узкую боль: «AI-ассистент по внутренней документации компании» (RAG с источниками).
- Реализуй: загрузка документов → RAG → ответы с цитатами → API/простой UI.
- Подготовь оценку качества на наборе реальных вопросов.
- Оформи: репозиторий с README (проблема → решение → метрики → как запустить), короткое демо-видео (2–3 мин), one-pager с описанием и ценой.

> [!success] Критерий зачёта (= готовность к Нише 1)
> - [ ] RAG отвечает по документам и всегда показывает источник.
> - [ ] Есть измеренное качество на ≥20 вопросах.
> - [ ] Доступ через API или простой UI.
> - [ ] Есть README-кейс + демо-видео + one-pager с ценой.
> - [ ] Можешь за 5 минут объяснить незнакомому человеку, какую проблему это решает и сколько стоит.
>
> Как только все пункты закрыты — переходи к Фазе 1 [[solo-monetization-plan|плана]] (первые клиенты).

## Что осознанно НЕ входит в курс (для соло на старте)
- Обучение фундаментальных моделей с нуля — это бюджет и команда.
- Глубокий MLOps/Kubernetes/распределённое обучение — избыточно для одиночки на старте.
- Теория глубокого обучения на уровне вывода формул — бери по мере необходимости, а не наперёд.

> [!note] Принцип
> Учись ровно до критерия зачёта каждого модуля и сразу применяй на капстоуне/реальной задаче. «Доучивание впрок» без выхода на клиентов — главная ловушка.
